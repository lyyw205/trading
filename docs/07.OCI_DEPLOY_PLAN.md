### 1-2. `docker/docker-compose.prod.yml` 생성

```yaml
version: "3.9"

services:
  app:
    image: ${REGISTRY_URL:-ghcr.io/iamooo}/trading-bot:${IMAGE_TAG:-latest}
    env_file:
      - /opt/trading/.env
    ports:
      - "127.0.0.1:8000:8000"
    depends_on:
      db:
        condition: service_healthy
    restart: unless-stopped
    logging:
      driver: json-file
      options:
        max-size: "10m"
        max-file: "3"

  db:
    image: postgres:16-alpine
    env_file:
      - /opt/trading/.env
    environment:
      POSTGRES_DB: ${POSTGRES_DB:-crypto_trader}
      POSTGRES_USER: ${POSTGRES_USER:-postgres}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
    volumes:
      - pgdata:/var/lib/postgresql/data
    # 포트 외부 노출 안 함 (컨테이너 네트워크 내부만)
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_USER:-postgres} -d ${POSTGRES_DB:-crypto_trader}"]
      interval: 10s
      timeout: 5s
      retries: 5
    restart: unless-stopped
    logging:
      driver: json-file
      options:
        max-size: "10m"
        max-file: "3"

volumes:
  pgdata:
```

### 1-3. `deploy.yml` 수정사항

```yaml
# 변경 1: Dockerfile 경로 명시
- name: Build Docker image
  run: docker build -f docker/Dockerfile -t trading-bot:${{ github.sha }} .

# 변경 2: 배포 스크립트에 마이그레이션 추가 + prod compose 사용
- name: Deploy to server
  uses: appleboy/ssh-action@v1
  with:
    host: ${{ secrets.DEPLOY_HOST }}
    username: ${{ secrets.DEPLOY_USER }}
    key: ${{ secrets.DEPLOY_KEY }}
    script: |
      cd /opt/trading

      # 레지스트리 로그인 + 이미지 pull
      echo "${{ secrets.REGISTRY_PASSWORD }}" | docker login ${{ secrets.REGISTRY_URL }} -u ${{ secrets.REGISTRY_USER }} --password-stdin
      docker pull ${{ secrets.REGISTRY_URL }}/trading-bot:latest

      # DB 마이그레이션
      docker compose -f docker-compose.prod.yml run --rm app alembic upgrade head

      # 앱 재시작
      docker compose -f docker-compose.prod.yml up -d --no-deps app

      # 헬스체크
      sleep 10
      curl -f http://localhost:8000/health || (docker compose -f docker-compose.prod.yml logs app --tail=50 && exit 1)

      # 이전 이미지 정리
      docker image prune -f
```

### 1-4. GitHub Secrets 설정 필요 목록

| Secret 이름 | 값 | 비고 |
|---|---|---|
| `DEPLOY_HOST` | OCI 인스턴스 퍼블릭 IP | |
| `DEPLOY_USER` | `ubuntu` 또는 `opc` | OCI 기본 유저 |
| `DEPLOY_KEY` | SSH 프라이빗 키 | |
| `REGISTRY_URL` | `ghcr.io/iamooo` | GHCR 사용 시 |
| `REGISTRY_USER` | GitHub 유저명 | |
| `REGISTRY_PASSWORD` | GitHub PAT (packages 권한) | |
| `DISCORD_WEBHOOK` | (선택) 배포 알림용 | |

## 2단계: OCI 서버 준비

### 2-1. 기존 프로젝트 정리

```bash
# SSH 접속
ssh -i ~/.ssh/oci_key ubuntu@<OCI_IP>

# 기존 프로젝트 확인
docker ps -a
docker compose ls

# 기존 프로젝트 중지 및 제거
cd /opt/기존프로젝트경로
docker compose down

# 기존 볼륨 확인 (필요한 데이터 있으면 백업)
docker volume ls

# 전체 정리 (미사용 이미지, 네트워크, 볼륨 제거)
docker system prune -a --volumes
# ⚠️ --volumes는 미사용 볼륨도 삭제하므로 백업 확인 후 실행

# 기존 디렉토리 제거
sudo rm -rf /opt/기존프로젝트경로
```

### 2-2. 서버 환경 구성

```bash
# Docker + Docker Compose 확인 (이미 설치되어 있을 것)
docker --version
docker compose version

# 프로젝트 디렉토리 생성
sudo mkdir -p /opt/trading
sudo chown $USER:$USER /opt/trading
cd /opt/trading

# .env 파일 생성
cat > .env << 'ENVEOF'
# ===== Database =====
DATABASE_URL=postgresql+asyncpg://postgres:STRONG_PASSWORD_HERE@db:5432/crypto_trader
POSTGRES_DB=crypto_trader
POSTGRES_USER=postgres
POSTGRES_PASSWORD=STRONG_PASSWORD_HERE

# ===== Initial Admin =====
# 관리자 생성: python scripts/create_admin.py --email admin@example.com

# ===== Encryption =====
# 생성: python3 -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
ENCRYPTION_KEYS=GENERATE_AND_PASTE_HERE

# ===== Session & CSRF =====
# 생성: python3 -c "import secrets; print(secrets.token_urlsafe(32))"
SESSION_SECRET_KEY=GENERATE_AND_PASTE_HERE
CSRF_SECRET=GENERATE_AND_PASTE_HERE

# ===== App =====
APP_HOST=0.0.0.0
APP_PORT=8000
DEBUG=false
LOG_LEVEL=INFO
ENVEOF

# .env 권한 제한
chmod 600 .env

# docker-compose.prod.yml 배치
# (CI/CD에서 scp로 전송하거나, 서버에 직접 생성)
# 내용은 위 1-2 참고
```

### 2-3. OCI 네트워크 보안 설정

OCI 콘솔 → Networking → Virtual Cloud Networks → Security Lists:

```
# 열어야 할 포트
Ingress Rules:
  - 22/TCP   (SSH)       ← 내 IP만 허용
  - 80/TCP   (HTTP)      ← 0.0.0.0/0
  - 443/TCP  (HTTPS)     ← 0.0.0.0/0

# 절대 열지 않을 포트
  - 5432 (PostgreSQL)    ← 차단 확인
  - 8000 (앱 직접)       ← 차단 (리버스 프록시 통해서만)
```

OCI 인스턴스 iptables도 확인:

```bash
sudo iptables -L INPUT -n
# 또는 Oracle Linux의 경우
sudo firewall-cmd --list-all
```

## 3단계: 최초 수동 배포 (CI/CD 전 1회)

```bash
cd /opt/trading

# 1. 레지스트리 로그인 (GHCR 예시)
echo "GITHUB_PAT" | docker login ghcr.io -u USERNAME --password-stdin

# 2. docker-compose.prod.yml이 있는지 확인
ls -la docker-compose.prod.yml

# 3. DB 먼저 기동
docker compose -f docker-compose.prod.yml up -d db
# DB 준비 대기
sleep 5
docker compose -f docker-compose.prod.yml exec db pg_isready

# 4. DB 마이그레이션
docker compose -f docker-compose.prod.yml run --rm app alembic upgrade head

# 5. 앱 기동
docker compose -f docker-compose.prod.yml up -d app

# 6. 확인
docker compose -f docker-compose.prod.yml ps
curl http://localhost:8000/health
docker compose -f docker-compose.prod.yml logs -f app
```

## 4단계: 리버스 프록시 (Caddy 권장)

```bash
# Caddy 설치 (Ubuntu)
sudo apt install -y debian-keyring debian-archive-keyring apt-transport-https
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | sudo gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | sudo tee /etc/apt/sources.list.d/caddy-stable.list
sudo apt update && sudo apt install caddy

# Caddyfile 설정
sudo tee /etc/caddy/Caddyfile << 'EOF'
your-domain.com {
    reverse_proxy localhost:8000
}
EOF

# 도메인 없이 IP만 사용할 경우
sudo tee /etc/caddy/Caddyfile << 'EOF'
:80 {
    reverse_proxy localhost:8000
}
EOF

sudo systemctl enable caddy
sudo systemctl restart caddy
```

## 5단계: 운영 유지보수

### DB 백업 크론잡

```bash
# 백업 스크립트 생성
sudo tee /opt/trading/backup-db.sh << 'SCRIPT'
#!/bin/bash
BACKUP_DIR=/opt/trading/backups
mkdir -p $BACKUP_DIR
docker compose -f /opt/trading/docker-compose.prod.yml exec -T db \
  pg_dump -U postgres crypto_trader | gzip > "$BACKUP_DIR/db_$(date +%Y%m%d_%H%M%S).sql.gz"
# 7일 이상 된 백업 삭제
find $BACKUP_DIR -name "*.sql.gz" -mtime +7 -delete
SCRIPT
chmod +x /opt/trading/backup-db.sh

# 크론 등록 (매일 03:00)
(crontab -l 2>/dev/null; echo "0 3 * * * /opt/trading/backup-db.sh") | crontab -
```

### 모니터링

```bash
# 간단한 헬스체크 크론 (5분마다)
(crontab -l 2>/dev/null; echo "*/5 * * * * curl -sf http://localhost:8000/health > /dev/null || (cd /opt/trading && docker compose -f docker-compose.prod.yml restart app)") | crontab -
```

### 로그 확인

```bash
docker compose -f docker-compose.prod.yml logs -f app --tail=100
docker compose -f docker-compose.prod.yml logs -f db --tail=50
```

## 서버에 존재하는 파일 요약

```
/opt/trading/
├── .env                        # 시크릿 (수동 관리, chmod 600)

├── backup-db.sh                # DB 백업 스크립트
└── backups/                    # DB 백업 파일들 (자동 생성)
```
