로그 레퍼런스 가이드

크립토 멀티트레이더가 운영 중 출력하는 모든 로그의 의미와 대응 방법

로그 읽는 법

모든 로그는 JSON 한 줄로 출력됩니다. 핵심 필드는 level(긴급도)과 msg(내용)입니다. account_id로 어떤 계정인지, cycle_id로 어떤 매매 사이클인지 추적할 수 있습니다. 

{"ts":"2026-03-03 12:00:00", "level":"WARNING", "account_id":"a1b2c3d4-...", "cycle_id":"abc123", "msg":"Balance check failed: timeout, skipping buy evaluation"}

긴급도 구분

위험 (CRITICAL)
거래가 완전히 중단됨. 즉시 확인 필요
오류 (ERROR)
특정 작업이 실패함. 반복되면 거래 중단 가능
주의 (WARNING)
일부 기능이 제한됨. 상황 지켜보기
알림 (INFO)
정상 작동 기록. 별도 조치 불필요

긴급 대응 가이드

즉시 대응 필요

  • Circuit breaker triggered
    → 계정 거래 완전 중단. API 키·네트워크 확인 후 수동 재시작
  • Trading engine crashed
    → 전체 서비스 다운. 서버 즉시 재시작 필요
  • Reconciliation drift
    → 거래소와 DB 불일치. 포지션·체결 수동 확인
  • place TP sell failed
    → 이익 실현 매도 불가. 수동으로 매도 주문 등록

30분 이내 확인

  • _init_client() failed
    → API 키 확인, 계정 설정 점검
  • Transient error (Nx)
    → 연속 오류 누적 중. 5회 도달 전 원인 파악
  • WS fatal error / failed to recreate
    → 실시간 가격 수신 중단. 네트워크 상태 확인
  • place LOT/TREND buy failed
    → 매수 주문 실패. 잔고·API 키 확인
  • Buy pause → PAUSED
    → 잔고 부족으로 매수 중단. 입금 고려
